# CLAUDE.md — Инструкции для Claude (MyKHSU Backend)

Этот файл содержит правила для Claude Sonnet / Claude Opus при работе с этим репозиторием.

> Общие правила: см. [AGENTS.md](AGENTS.md)

---

## Приоритеты при работе

1. Сначала прочитай изменяемый файл, чтобы понять контекст
2. Вноси только **необходимые** изменения — не рефакторь то, о чём не просили
3. После изменения кода **обновляй документацию** из таблицы в AGENTS.md

## Стиль кода

- **Язык:** JavaScript ESM, Node.js 20+
- **Стиль:** без точек с запятой в конце строки — не требуется, но будь последовательны с существующим кодом
- **Комментарии:** на русском языке минимальными вставками — только там, где логика неочевидна
- **Именование:** camelCase для переменных и функций Express, PascalCase для React-компонентов

## Работа с базой данных

- `pairsDb` — для таблиц `pairs`, `times`, `news`
- `usersDb` — для таблицы `users`
- Всегда используй prepared statements: `db.prepare('SELECT ... WHERE x = ?').get(value)`
- Для транзакций: `db.transaction(fn)(args)`

## Работа с React Admin Panel

Файлы в `admin-panel/src/styles.css` применяют CSS-переменные MyKHSU-стиля:
- `--accent` — акцентный цвет
- `--background` — фон страницы
- `--surface-card` — фон карточки
- `--text-primary` / `--text-secondary` / `--text-muted` — тексты
- `--border` — граница

Новые компоненты должны использовать только существующие CSS-переменные, не добавляя захардкоженные цвета.

## Контекст взаимодействия Claude

Когда пользователь просит добавить новый API-эндпоинт:
1. Добавь handler в `src/routes/user.js` (публичный) или `src/routes/admin.js` (защищённый)
2. Добавь описание в `docs/api.md`
3. Если нужна новая таблица или колонка — обнови `src/db/database.js` и `docs/database.md`

Когда пользователь просит добавить экран в Admin Panel:
1. Создай `admin-panel/src/screens/NewScreen.js`
2. Добавь маршрут в `admin-panel/src/App.js`
3. Добавь пункт в сайдбар (компонент `<nav>` в `App.js`)
4. Обнови таблицу экранов в `docs/frontend.md`
